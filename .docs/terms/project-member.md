# プロジェクトメンバー

## 定義

プロジェクトに参加するユーザーを指します。各プロジェクトメンバーは、プロジェクトとユーザーの関連付け、および特定の役割（ロール）によって定義されます。

## 例

- 田中さんが「マーケティングプロジェクト」のオーナーとして登録されている
- 山田さんが「開発プロジェクト」のメンバーとして招待された

## 補足説明

プロジェクトメンバーは以下のいずれかのロールを持ちます：

1. オーナー (OWNER): プロジェクトの作成者。すべての権限を持ち、プロジェクトの削除を含む全操作が可能。
2. 管理者 (ADMIN): プロジェクトの設定変更、メンバー管理などの管理権限を持つ。
3. メンバー (MEMBER): プロジェクトへの参加と基本的な操作権限を持つ。
4. 閲覧者 (VIEWER): プロジェクト内容の閲覧のみ可能で、変更権限は持たない。

ロールによって、プロジェクト内でのアクセス権限と操作可能な機能が決定されます。
